# Phase 43 — Reconcile the streaming-idempotency claims to the message-granular seam (R-P61J-IHKB, R-Y878-6UDJ, R-6YLW-9NXA)

*Realizes design Decision 11 (retry & backoff policy — the "no-retry after the first delivered byte" family). Depends on Phase 06 (the retry policy), Phase 24 (the migration to message-granular delivery), Phase 13 (MCP `tools/call` integration), and Phase 31 (the shared `internal/retry` executor).*

Close audit Finding B / cross-cutting theme 1 (`project/audit/findings.md`, section B and "Cross-cutting themes"; analysis in `project/audit/b-section-fixes.md`): three D11 claims were written as "a failure **after a byte has been delivered** to the consumer is never retried." That phrasing is **structurally unfalsifiable** at the current seam. Since the migration to message-granular delivery (Decision 2, Phase 24), each round-trip is assembled whole on the backend and the orchestrator forwards events to the consumer `Stream` only *after* the round-trip returns error-free (orchestration.go:306-335); a failed round-trip therefore delivers **no** events (already pinned by R-CDQ0-EM4Z). The MCP path is identical: `CallTool` assembles the entire JSON-RPC response before returning, so a tool-result is never partially delivered. The "byte-delivered-then-failed" state the original wording guards against cannot occur — so there is nothing to falsify, and as written `R-P61J-IHKB`'s test actually asserts the *inverse* (a partial-message-bundled-with-error round-trip **is** retried).

This is a **design-vs-test reconciliation**, not merely a missing assertion, and it is resolved by correcting the claim wording — *not* by re-introducing an incremental-delivery seam (Decision 2 deliberately rejected delta delivery; adding a partial-delivery state purely to make the old wording testable would undo that decision and re-open the early-break/goroutine-leak surface it closed). The D11 design wording has already been rewritten in place in this change set: the settled choice now states the round-trip is the retry unit *enforced by construction* via whole-round-trip buffering, and the three Verification claims now assert the structurally-true, falsifiable guarantees:

- **R-P61J-IHKB** — a failed round-trip's partial output is discarded and the whole round-trip re-issued from the same idempotent `Request` and re-streamed from scratch, with no partial or double delivery to the consumer.
- **R-Y878-6UDJ** — the retry budget (attempts + backoff) is independent per round-trip, and eligibility depends only on the round-trip's category and its own budget, never on what earlier round-trips delivered.
- **R-6YLW-9NXA** — a `tools/call` transport failure is surfaced without automatic retry, and its result is assembled whole before feedback, so no partial tool-result ever reaches the loop.

End state:

- Each of the three id-carrying tests (`retry_test.go` for R-P61J-IHKB and R-Y878-6UDJ; `mcp_integration_test.go` for R-6YLW-9NXA) is confirmed — and renamed/re-commented as needed — to assert the reworded by-construction guarantee, not the old "after a byte" wording. R-P61J-IHKB's test asserts the failed round-trip is re-issued whole and the consumer sees exactly the retried message once (no double/partial delivery); R-Y878-6UDJ's test keeps its per-round-trip-budget assertion (`provider.calls == 4`, two independent backoffs); R-6YLW-9NXA's test keeps its no-retry / whole-result assertions.
- A note (comment or shared assertion) ties the family to R-CDQ0-EM4Z — "a failed round-trip delivers no events" — as the structural fact that makes the byte-delivered state unrepresentable.
- No production change: the guarantee is already provided by the buffering architecture; this phase aligns the claims and tests with how it is actually enforced.

**Done when:** R-P61J-IHKB, R-Y878-6UDJ, and R-6YLW-9NXA are each covered by a clearly-named test that asserts the reworded structural guarantee (round-trip / `tools/call` as the whole-buffered retry unit; per-round-trip budget independence; no partial or double delivery), consistent with the updated D11 wording; the other D11 ids stay green; and the suite is green per design Conventions.
