# Phase 21 — Native reasoning introspection surface: descriptor types and per-package specs
*Realizes design Decision 16 (the reasoning-introspection portion: `ReasoningKind`/`ReasoningSpec`/`Sentinel`/`ReasoningInspector`, each sub-package's credential-blind `Reasoning` value, and the baked-in reasoning-spec table) and Decision 6 (the `ReasoningValue` value model and the `WarningCode` taxonomy, introduced here as type declarations). Depends on Phase 1 (root value/`Warning` types) and Phases 8–11 (each provider's existing model registry, which the spec data is co-located with).*

This phase is **purely additive** — `GenSettings.Reasoning` keeps the old `ReasoningEffort` type and no adapter lowering changes, so the build stays green throughout. Root `agentkit` gains the credential-blind introspection surface: `ReasoningKind` (`ReasoningEnum`/`ReasoningRange`/`ReasoningToggle`), `ReasoningSpec{Term,Kind,Levels,Min,Max,Sentinels,Default,CanDisable}`, `Sentinel{Value,Meaning}`, and the `ReasoningInspector` interface (`ReasoningSpec(model)(ReasoningSpec,bool)` and `SupportedReasoning() map[string]ReasoningSpec`). The opaque `ReasoningValue` carrier and its three constructors (`Level`/`Budget`/`DisableReasoning`) are introduced here because `ReasoningSpec.Default` is a `ReasoningValue`; the four states (unset, native level, native budget, explicit-disabled) are mutually exclusive by construction (unexported fields), and a `spec.accepts(ReasoningValue) bool` predicate (the warn-and-default decision of D6, pure spec logic) lives on the root type. `Warning` gains a `Code WarningCode` field, and the `WarningCode` enum is declared (`WarnReasoningUnsupported`, `WarnReasoningCannotDisable`, `WarnToolChoiceForced`, `WarnToolSchemaLossy`); because the zero value is `WarnReasoningUnsupported`, the two existing non-reasoning warning sites (Z.ai forced `tool_choice`, Gemini/MCP schema-lossy) are tightened to set their explicit `Code` so they can never be misread as a reasoning warning. Each provider sub-package (`anthropic`, `google`, `openai`, `zai`) gains a package-level `var Reasoning agentkit.ReasoningInspector` backed by a `ReasoningSpec` added to every reasoning model's existing registry entry, co-located with the model constant and copied verbatim from the design's reasoning-spec table — reading only static registry data, constructing no `Provider` handle, issuing no network call. (Per the per-phase introspection scope, all four providers' specs land together here, so the cross-provider introspection ids are fully discharged in this one phase.)

**Done when:** R-S6NB-RYUE (each `Reasoning` introspector returns specs with no handle/network — credential-blind), R-S7V8-5QL3 (`SupportedReasoning()` returns a spec per reasoning-capable model, keyed by id), R-S934-JIBS (`ReasoningSpec(model)` is `(spec,true)` for a known id, `(_,false)` for an unknown one), R-EN2N-9B9F (each registry's per-model `ReasoningSpec` equals the design's reasoning-spec table), R-EPIG-0UQT (every model's `spec.Default` is itself accepted by that model's own `spec`), and R-T6G3-NJ7L (the four `ReasoningValue` states are mutually exclusive — a value built by one constructor never reads as another) are covered; the package compiles with `GenSettings.Reasoning` still `ReasoningEffort`; suite green.
