# Decision 2 — The consumption surface: `Stream` and the message-granular `Event` taxonomy

**Decision.** `Send` returns a `*Stream` the consumer drains exactly once. Every observable thing in a turn — a completed assistant message, a tool call the model requested, the tool result AgentKit fed back, the final usage — arrives as a variant in one ordered event stream. The consumer learns one loop and one type switch, regardless of provider. This is where the "full transparency" promise lands and where reasoning and tool use are made to look uniform: they are simply event variants — or, for a model's visible text and reasoning summary, fields of the completed `Message` that a `MessageDone` carries.

**Delivery is message-granular, not token-by-token.** The smallest unit the consumer observes is a *completed* unit: a whole assistant message, a whole tool call, a whole tool result. AgentKit does **not** surface sub-message (token) fragments to the consumer. A model's visible answer text and its reasoning summary are not streamed as deltas; they ride, complete, inside the `Message` delivered by `MessageDone` (visible text as `TextBlock`s, the reasoning summary as `ReasoningBlock.Summary`). The provider's incremental SSE wire format is still consumed on the backend to assemble each round-trip's message (Decision 9, Decision 12) — that decoding simply terminates at the adapter and is not propagated upward as consumer events.

`Stream` follows the `sql.Rows`/`bufio.Scanner` shape on Go 1.23+ range-over-func:

```go
// Stream is the result of one Send. Drained exactly once, by one goroutine.
// Iterating to completion (or breaking early) releases resources.
type Stream struct { /* unexported */ }

// Events yields each event of the turn in order until the turn completes or
// fails. Breaking early is safe: it runs cleanup (closes the HTTP body).
func (s *Stream) Events() iter.Seq[Event]

// Err returns the terminal error of the turn, or nil. Valid only after Events
// has been fully consumed (cf. bufio.Scanner.Err).
func (s *Stream) Err() error

// Usage returns the token usage for the turn. Valid only after Events has been
// fully consumed.
func (s *Stream) Usage() Usage

// Warnings returns any settings the provider could not honor as asked and
// degraded (Decision 6). Valid after Events is fully consumed; empty when
// everything was honored.
func (s *Stream) Warnings() []Warning
```

Consumer loop:

```go
stream := conv.Send(ctx, "What time is it in Tokyo?")
for ev := range stream.Events() {
    switch ev := ev.(type) {
    case agentkit.MessageDone: // one assistant message completed (visible text + any reasoning summary ride inside ev.Message)
    case agentkit.ToolUse:     // model asked to use a tool
    case agentkit.ToolResult:  // AgentKit ran it, fed the result back
    }
}
if err := stream.Err(); err != nil { /* typed, classifiable */ }
usage := stream.Usage()
```

The `Event` taxonomy is a sealed interface (idiomatic Go tagged union; type-switch dispatch). Every variant is a whole, semantically-complete unit — there are no sub-message deltas:

```go
type Event interface{ isEvent() }

// Whole, semantically-complete events:
type ToolUse    struct{ ID, Name string; Input json.RawMessage }       // model requested a tool
type ToolResult struct{ ID, Name string; Output string; IsError bool } // AgentKit's loop fed this back

// Boundary marker — one assistant message in the exchange has completed:
type MessageDone struct{ Message Message } // the full assembled message (visible TextBlocks, tool_use blocks, reasoning summary)
```

Settled choices:
- **`iter.Seq[Event]` + terminal `Err()`/`Usage()`**, not `iter.Seq2[Event, error]` and not a channel: one stream error is terminal (per-event errors are awkward), `Seq2` can't carry setup/teardown errors, channels leak goroutines on early `break`. Range-over-func gives clean early-exit cleanup.
- **Every event is a whole unit; there are no `*Delta` events.** Visible text and the reasoning summary are delivered complete, inside the `Message` a `MessageDone` carries — never as token fragments. The provider asymmetry (Anthropic/OpenAI stream partial text and partial tool-JSON, Gemini sends them whole) is absorbed *below* this surface: the adapter assembles each round-trip's full message from the SSE stream and the consumer always sees a complete `ToolUse` and a complete `Message`, never partial JSON or partial text. This is the design realization of the product's message-granular delivery promise.
- **`ToolResult` is an event** even though AgentKit (not the model) generated it — the consumer watches the auto-loop without driving it.
- **`MessageDone` carries the assembled `Message`** — the seam between the stream and `History`: each completed assistant message is both emitted here and appended to `History`. A consumer that wants the turn's visible answer reads it from `MessageDone.Message` (concatenating its `TextBlock`s); the model's reasoning summary, when present, is `MessageDone.Message`'s `ReasoningBlock.Summary`.
- **`ctx` is the one from `Send`**, checked inside iteration; cancelling it ends the stream with a context error in `Err()`.
- **Breaking out of `Events()` early abandons the turn.** Because delivery is message-granular, an early `break` happens at a message boundary (e.g. between the assistant message that requested tools and the next round-trip). It releases resources (R-CCI4-0UEA) and, because the turn did not complete, rolls back any `History` mutations (Decision 10 atomicity) and clears the `Conversation`'s "stream live" flag — so `History` reflects only completed turns and the next `Send` is unblocked.

**Rejected.**
- *Incremental (token-by-token) delivery to the consumer* — the prior design streamed `TextDelta`/`ReasoningDelta` fragments as the model generated. Dropped: the product no longer promises sub-message delivery (product.md, "Message-granular delivery"), and surfacing live deltas to the consumer would require holding the provider's HTTP body open across consumer iteration, re-arming the goroutine/early-break leak surface that R-CCI4-0UEA now satisfies by construction. The backend still decodes the provider's incremental SSE stream — only the consumer-facing fragment events are removed.
- *`iter.Seq2[Event, error]`* — per-event error that's really terminal; can't express setup/teardown failures.
- *`chan Event`* — goroutine leak on early break, `select` plumbing, no clean teardown hook.
- *Callbacks* — hide tool-call assembly, lose composability and early-exit.
- *A single `Delta` event with a `Kind` field* — moot now that no delta events exist; even when they did, distinct sealed types beat stringly-typed branching.
- *Surfacing the raw reasoning replay payload as an event* — the opaque signature/encrypted blob is preserve-and-replay-only; it rides in the message model (`ReasoningBlock.Opaque`), never the consumer event stream. Only the human-readable reasoning summary reaches the consumer, and only inside `MessageDone.Message`.

**Verification.**
- R-HUZX-7N2W — draining a text-only turn yields exactly one `MessageDone` whose `Message` carries the full assembled answer text (concatenated `TextBlock`s) and no sub-message (`*Delta`) events of any kind; after iteration ends `Err()==nil` and `Usage()` is populated.
- R-C8UE-VJ67 — a turn where the model requests a tool surfaces, in order, a `ToolUse` with assembled complete `Input` (never partial JSON) followed by the `ToolResult` AgentKit fed back, on every provider — including Gemini (whole-JSON) and Anthropic/OpenAI (fragmented-JSON assembled on the backend).
- R-CBA7-N2NL — each completed assistant message is emitted as a `MessageDone` carrying the fully assembled `Message` (visible text, tool_use blocks, and any reasoning summary), and that same message is what landed in `History`.
- R-CCI4-0UEA — breaking out of the `Events()` loop early releases the turn's resources (HTTP body closed) without leaking a goroutine.
- R-CDQ0-EM4Z — a failed turn surfaces its error only through `Err()` after iteration ends (events stop, no panic); `Err()` is nil on success.
