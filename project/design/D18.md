# Decision 18 — The embeddings consumer surface: the `Embedder` object and the `Embed` verb

**Decision.** Embeddings get their **own** consumer object — an `Embedder` — and **one** verb — `Embed` — that returns `(*EmbedResult, error)` in plain idiomatic Go. The `Embedder` is the embeddings analogue of `Conversation` (D1): a bundle of configuration the consumer mutates between calls, threaded explicitly into each call. It is deliberately *not* a `Conversation` and shares none of the conversational machinery — no `System`, `Tools`, `History`, `Gen`/reasoning, no `Stream`, no tool loop — because embeddings are a single, stateless request→response. The shared *foundations* (the D7 error model, the D11 `RetryPolicy`) are reused by value; the chat *surface* is not.

```go
package agentkit

// Embedder is a configured embeddings client: provider, model, and the options
// that shape every Embed call, plus the cumulative usage/cost it accrues over
// its lifetime. Like Conversation it is a plain struct literal the consumer
// mutates between calls (switching provider/model/dimension is field
// assignment), and like Conversation it is owned by one goroutine and is not
// safe for concurrent use (no internal locking).
type Embedder struct {
    Provider   EmbeddingProvider // swappable between calls; carries its own credentials (D19). OpenAI or Google only.
    Model      string            // swappable between calls; validated against the embeddings registry (D20) at the Embed boundary
    Dimensions int               // 0 → the model's native dimension; non-zero → a per-model target dimension, honored-or-failed-loud
    Retry      RetryPolicy       // reused verbatim from D11; zero value = defaults

    // cumulative accounting (unexported); read via TotalUsage()/TotalCost()
}

// InputType is the per-call query-vs-document role. It is an Embed argument, not
// an Embedder field, because the same Embedder legitimately embeds both queries
// and documents (D19/research §12.3 — it cannot be config). The zero value is
// InputUnspecified, so a caller that does not care passes it (or InputUnspecified)
// and every provider behaves correctly: Google maps it to a task_type, OpenAI
// ignores it (D19).
type InputType int
const (
    InputUnspecified InputType = iota // default; no role hint
    InputQuery                        // a search query (Google: RETRIEVAL_QUERY)
    InputDocument                     // a stored document (Google: RETRIEVAL_DOCUMENT)
)

// Embed turns inputs into one unit-normalized vector per input, in input order.
// inputs is a batch of any size (a single text is a batch of one); AgentKit
// chunks it across the provider's per-call limit internally (D19). role tags the
// whole batch. Configuration is validated at this boundary before any provider
// call; an unusable Embedder, a bad dimension, or empty input fails here.
//
// On success err is nil and *EmbedResult is non-nil. On failure *EmbedResult is
// nil and err carries the uniform D7 error (errors.Is for the category, errors.As
// for a provider *Error). This is the one place the embeddings surface departs
// from chat's *Stream: there is no stream to thread, so the terminal error is an
// ordinary return value rather than EmbedResult.Err().
func (e *Embedder) Embed(ctx context.Context, inputs []string, role InputType) (*EmbedResult, error)

// EmbedResult is one Embed call's result. Vectors[i] is the unit-normalized
// embedding of inputs[i], same order, same length as inputs. Warnings are
// non-fatal advisories (never a reason err is non-nil). Usage/Cost are this
// call's figures (summed across internal chunks, D19/D20).
type EmbedResult struct {
    Vectors  [][]float32
    Warnings []Warning      // reused from chat; non-fatal
    usage    EmbeddingUsage // D20
    cost     Cost           // D20 (shared root Cost type)
}

func (r *EmbedResult) Usage() EmbeddingUsage // this call's input-token usage (D20)
func (r *EmbedResult) Cost() Cost            // this call's dollar cost — always available (D20)

// Cumulative accounting over the Embedder's life, mirroring Conversation's
// TotalUsage()/TotalCost() (D15/D16) — the sum of every successful Embed call.
func (e *Embedder) TotalUsage() EmbeddingUsage
func (e *Embedder) TotalCost() Cost
```

Construction and a provider/model/dimension switch are plain field work, exactly like `Conversation`:

```go
emb := &agentkit.Embedder{Provider: openai.NewEmbedder(apiKey), Model: openai.EmbedModel3Small}
res, err := emb.Embed(ctx, []string{"hello", "world"}, agentkit.InputDocument)
// switch provider/model between calls — only configuration changes:
emb.Provider = google.NewEmbedder(geminiKey); emb.Model = google.EmbedModelGemini001; emb.Dimensions = 1536
qres, err := emb.Embed(ctx, []string{"a search query"}, agentkit.InputQuery)
```

Settled choices:
- **A separate `Embedder` object, not an overloaded `Conversation`.** Embeddings have no system prompt, tools, history, reasoning, streaming, or tool loop; folding them into `Conversation` would carry a dozen meaningless fields and a `Send` that means something different. The two objects share *foundations* (`RetryPolicy`, the D7 errors, the root `Cost`/`Warning` types) by value, not the chat surface (research §12.5–12.6).
- **`Embed(...) (*EmbedResult, error)` — idiomatic Go, not a `*Stream`.** Chat threads everything through `*Stream` *because it streams* message-granular events; an embeddings call has nothing to stream — it is one request→response. Returning a bare `(*EmbedResult, error)` is the honest shape; the D7 error model is unchanged (`errors.Is`/`errors.As` over the same sentinels and `*Error`). Forcing a fake single-event `Stream` onto a non-streaming call would cargo-cult the chat shape for symmetry's sake.
- **`Dimensions` is an `Embedder` field (per-model config); `InputType` is an `Embed` argument (per-call).** Research §12.3 is explicit: the query/document role *cannot* be config — one `Embedder` embeds both queries and documents, so the role rides the call. The target dimension *is* per-model config and changes rarely, so it is a field. `Dimensions == 0` means the model's native dimension; a non-zero value is honored-or-failed-loud against the model's producible set (D20).
- **Batch in, ordered vectors out; a single text is a batch of one.** `inputs []string` is the only input shape — no separate single-string verb on the surface (single-surface principle, D1). `Vectors[i]` corresponds to `inputs[i]`; AgentKit absorbs each provider's per-call size limit internally (D19), so the same call works regardless of provider or batch size.
- **Validation at the `Embed` boundary, reusing the D1/D7 sentinels — no new ones.** A missing `Provider`/`Model`, an unknown model id, or a `Dimensions` value the selected model cannot produce all surface as **`ErrInvalidConfig`** before any provider call (the dimension case is *fail-loud*: AgentKit never silently returns a different size). An empty batch — a nil/empty slice, or a batch containing an empty string — surfaces as **`ErrInvalidInput`**. An over-length single input surfaces as **`ErrContextLength`** from the provider (D19). The whole embeddings surface reuses the D7 taxonomy unchanged, honoring the product's uniform-error promise.
- **Per-call and cumulative accounting, mirroring chat.** `EmbedResult.Usage()/Cost()` are this call's figures; `Embedder.TotalUsage()/TotalCost()` accumulate across the Embedder's life — the embeddings analogue of `Stream`/`Conversation` accounting (D15/D16). Cost is **always** available (every supported embedding model is priced by construction, D20).
- **No embeddings event log in v1.** The chat `Log io.Writer`/`Close()`/`summary`/`ErrClosed` lifecycle (D15) is deliberately **not** mirrored: the product promises cumulative cost/usage (delivered by the accessors above) but does not promise an embeddings event trace, and a single-call request→response has no multi-event protocol stream worth serializing. Deferred, not rejected.

**Rejected.**
- *Overloading `Conversation`/`Send` for embeddings* — carries conversational fields that are meaningless for embeddings and a verb whose semantics fork; two objects keep each surface honest.
- *Mirroring `*Stream` with `EmbedResult.Err()` and a fake single event* — symmetry for its own sake; there is nothing to stream, and `(*EmbedResult, error)` is the idiomatic Go contract.
- *`InputType` as an `Embedder` field* — would force a consumer who embeds both queries and documents to hold two `Embedder`s, the exact two-client trap research §12.3 calls out.
- *A separate single-string `EmbedOne` verb* — violates the single-surface principle; a batch of one already covers it, and the consumer indexes `Vectors[0]`.
- *A new embeddings-specific error sentinel for the unsupported-dimension / over-length cases* — the D7 taxonomy already has `ErrInvalidConfig` and `ErrContextLength`; a new sentinel would fracture the uniform error surface for no gain.

**Verification.**
- R-Y5RV-WB3T — an `Embed` call returns exactly one vector per input in input order, and the identical calling code produces a working result against both OpenAI and Google (config-only swap holds at the surface; the only change between the two runs is `Provider`/`Model`).
- R-Y6ZS-A2UI — changing `Provider`/`Model` (and `Dimensions`) between two `Embed` calls runs the second call against the newly selected provider/model with no other change to the calling code.
- R-Y87O-NUL7 — `Embed` on an `Embedder` missing a required config field (provider or model), or naming a model id the provider's embeddings registry does not know, returns `ErrInvalidConfig` (matchable via `errors.Is`) and issues no provider call.
- R-Y9FL-1MBW — `Embed` with empty input — a nil/empty `inputs` slice, or a batch containing an empty string — returns `ErrInvalidInput` (matchable via `errors.Is`) and issues no provider call.
- R-YANH-FE2L — `role` is an `Embed` argument defaulting to `InputUnspecified` (the zero value); the same call succeeds on both OpenAI and Google for each of `InputUnspecified`/`InputQuery`/`InputDocument` (the provider-side effect of the role is D19's concern).
- R-YBVD-T5TA — with `Dimensions == 0`, returned vectors have the model's native dimension; with `Dimensions` set to a value the selected model can produce, every returned vector has exactly that length.
- R-YD3A-6XJZ — with `Dimensions` set to a value the selected model cannot produce, `Embed` returns `ErrInvalidConfig` (fail-loud) and issues no provider call — it never silently returns vectors of a different size.
- R-YFJ2-YH1D — `EmbedResult.Usage()`/`Cost()` report this call's figures, and `Embedder.TotalUsage()`/`TotalCost()` equal the running sum of every successful `Embed` call's usage/cost over the Embedder's life.
