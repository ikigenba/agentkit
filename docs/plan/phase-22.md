# Phase 22 — Native reasoning value and warn-and-default lowering: atomic retype across root and all adapters
*Realizes design Decision 6 (the native `ReasoningValue` consumer surface and the build-time warn-and-default choke point) and Decision 9 (each adapter's native lowering keyed on the model's `spec.Kind`). Depends on Phase 21 (the per-model specs and `spec.accepts` it consumes), Phase 5 (the SPI `Request`/`GenSettings` seam and orchestration), and Phases 8–11 plus Phase 10's `internal/openaicompat` (the five lowering sites it reworks).*

The native-first retype is **atomic**: the instant `GenSettings.Reasoning` changes from `ReasoningEffort` to `agentkit.ReasoningValue`, all five lowering sites stop compiling against the old ordinal, so they are migrated together in this one phase and the build is green only at its end (no transitional two-field shim — principles: delete over maintain compat). `GenSettings.Reasoning` is retyped to `ReasoningValue` and the `ReasoningEffort` ordinal plus its `Effort*` constants are deleted. All five lowering sites — `anthropic.applyReasoning`, the `openai` effort path (`openAIReasoningEffort` and its request guard), `google.thinkingConfig` (`thinkingLevel`/`thinkingBudget`), and `internal/openaicompat.applyReasoning` — are reworked from the old ordinal mapping to native lowering keyed on the selected model's `spec.Kind`, exactly per the Decision 9 lowering table: a `Level` reaches the wire as the model's native effort/level field (Anthropic opus/sonnet also enabling thinking via `type:"adaptive"`), a `Budget` as the native token-budget field (haiku clamped to `min(spec.Max, MaxTokens−1)`; Gemini never sending both `thinkingBudget` and `thinkingLevel`), `DisableReasoning()` as the model's native off-form, and unset omitting all reasoning parameters. The single build-time warn-and-default choke point lives in each adapter against `req.Model`: `if !spec.accepts(req.Gen.Reasoning) { apply spec.Default; emit one Warning }`, covering every non-native case (wrong value kind, out-of-range budget or bad level, can't-disable, and a value carried over from a previously-selected model) — never silent, never turn-breaking, the request still issued with the model's default.

**Done when:** R-T40A-VZQ7 (a `Level`/`Budget`/`DisableReasoning` reaches the adapter as the exact native form it was constructed with, no cross-model mapping), R-T587-9RGW (an unset `Reasoning` omits all reasoning parameters and emits no `Warning`), R-B7YX-J342 (a non-native value is replaced by `spec.Default` in the built request and records exactly one `Warning{Setting:"reasoning",Code:WarnReasoningUnsupported}`, the turn still succeeding), R-B96T-WUUR (validation runs at request-build against `req.Model` — a value native to a previously-selected model but invalid for the now-selected one warns and applies the new model's default), R-ELUQ-VJIQ (each adapter lowers per the Decision 9 table and a Gemini request never carries both budget and level), and the native re-proof of R-P89V-WVXD (a `DisableReasoning()` against a can't-disable model — e.g. gpt-5.5-pro, gemini-2.5-pro — falls back to the default and records `Warning{Code:WarnReasoningCannotDisable}`, turn still succeeding) are covered; the build compiles with `GenSettings.Reasoning` now `ReasoningValue` and `ReasoningEffort` gone; suite green.
