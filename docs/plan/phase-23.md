# Phase 23 — Fix: Anthropic replayed thinking block serializes its reasoning text in `text` instead of `thinking`
*Realizes design Decision 9 (the Anthropic thinking-block replay-serialization correction and R-TQ77-6QLK). Depends on Phase 22 (the native lowering that enables thinking on Anthropic — `Level`→`type:"adaptive"`, `Budget`→`type:"enabled"` — so a `ReasoningBlock` exists to replay); the Anthropic sibling of the OpenAI (Phases 16–17) and Gemini (Phases 19–20) replay-shape traps.*

Fix a defect in the `anthropic` adapter's request build (`anthropic/anthropic.go`, `convertMessage`): a replayed assistant `ReasoningBlock` serializes its reasoning text into the block's `text` field instead of the `thinking` field Anthropic's Messages API requires. The `ReasoningBlock` case builds `wireBlock{Type:"thinking", Text: b.Summary, Signature: sig}`, and `wireBlock` has no `thinking` field at all, so the block ships as `{"type":"thinking","text":<text>,"signature":<sig>}` — missing the required `thinking` string (the text the `signature` signs). The first turn has no prior thinking block to resend and succeeds, but the **second** turn of any reasoning-enabled Anthropic conversation — the first to serialize a `thinking` input block (it lands at `messages[1].content[0]`, before the assistant's text) — fails with `400 messages.N.content.0.thinking.thinking: Field required`. Because an Anthropic `Level` enables thinking (`type:"adaptive"`) and a `Budget` enables it directly, the trap fires for every reasoning-on Anthropic model (opus-4-8 / sonnet-4-6 and haiku-4-5). The `signature` round-trips correctly (captured into `ReasoningBlock.Opaque`); the gap is purely the outbound field name the summary lands in, which no existing wire-shape test asserts — the byte-for-byte `Opaque` round-trips (R-IN0J-QMSI / R-XW08-D4YL) check the signature, not the JSON field, which is the gap that let this through. The parse side is unaffected (Anthropic streams the summary in via `thinking_delta`, not the `text` field), so this is a pure outbound build defect, the same shape as the OpenAI/Gemini replay traps. Observed via agentrepl (`-c provider=anthropic -c model=claude-haiku-4-5 -c thinking_budget=2048`): a clean first reply, then `anthropic: messages.1.content.0.thinking.thinking: Field required (status=400)` on the next turn. End state: `wireBlock` gains a `thinking` field (`Thinking string` tagged `json:"thinking,omitempty"`) and the `ReasoningBlock` case sets `Thinking: b.Summary` rather than `Text: b.Summary`, emitting `{"type":"thinking","thinking":<text>,"signature":<sig>}`. A wire/replay test exercising a reasoning-turn follow-up pins the regression: it asserts the rebuilt request body's thinking block carries a non-empty `thinking` field (and no `text` field for that block) and that the round-trip does not surface `400`/`ErrInvalidRequest`. (Anthropic-specific — OpenAI/Gemini/Z.ai replay reasoning via their own input-item shapes and are unaffected.)

**Done when:** R-TQ77-6QLK is covered by a clearly-named test (Anthropic wire/replay exercising a reasoning-turn follow-up round-trip, asserting the replayed thinking block serializes its text in the `thinking` field — not `text` — and no `400`/`ErrInvalidRequest`); suite green.
