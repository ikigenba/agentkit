# Phase 16 — Fix: OpenAI replayed reasoning item missing the required `summary` field
*Realizes design Decision 9 (the OpenAI Responses reasoning-replay correction). Depends on Phase 9.*

Fix a defect in the `openai` adapter's request build (`openai/openai.go`): a replayed `reasoning` **input** item omits the `summary` field whenever `ReasoningBlock.Summary` is empty, because the shared `inputItem.Summary` field is tagged `json:"summary,omitempty"` and `openAIReasoningItem` only sets it when the summary is non-empty. The adapter never requests summaries (it sends `reasoning.effort` only, no `summary:"auto"`), so `Summary` is empty on essentially every reasoning item — and OpenAI's Responses API requires every reasoning input item to carry a `summary` array (empty `[]` allowed, but present). The result: the **second** turn of every OpenAI reasoning conversation — the first turn that replays a prior reasoning item (it lands at `input[1]`) — fails with `400 Missing required parameter: 'input[N].summary'`. The first turn succeeds (nothing to replay), so single-turn tests miss it; observed via agentrepl (`-c provider=openai`): the turn after the model's first reasoning reply errors out before any provider work. End state: a replayed reasoning item always serializes `summary` — `[]` when empty, `[{"type":"summary_text","text":…}]` when not — with the empty case surviving on the wire (the `omitempty` trap removed for the reasoning item, e.g. via a dedicated reasoning input-item shape so non-reasoning items do not grow a spurious `summary` key). A golden/replay test whose first turn produces a reasoning item with **no** summary pins the regression: it asserts the second request body's reasoning item contains `"summary":[]` and that the round-trip does not surface `ErrInvalidRequest`/`400`. (Requesting `summary:"auto"` is out of scope here — it is a separate transparency follow-up and does not by itself fix the 400, since auto-summaries can still come back empty and must replay `summary:[]`.)

**Done when:** R-OMKB-AY19 is covered by a clearly-named test (OpenAI golden/replay exercising the empty-summary reasoning replay, asserting `"summary":[]` on the wire and no `400`/`ErrInvalidRequest`); suite green.
