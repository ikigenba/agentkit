# Phase 25 — Cover the Anthropic in-stream SSE `error` event (200-then-`event: error`)

*Realizes design Decision 7 (the error model — the newly-minted in-stream classification id R-FR35-46U7). Depends on Phase 08 (the Anthropic adapter, its SSE parse, and the fake-server / golden-SSE harness this test is built on).*

Close audit Finding 7: the Anthropic adapter's in-stream error path (`anthropic/anthropic.go`, `classifyStreamError` at the `frame.Type == "error"` branch) has zero test coverage. It is the **only** path for a provider overload / rate-limit signaled **mid-stream on an HTTP 200** response: the request succeeds, the `text/event-stream` opens, and an `event: error` frame arrives carrying `{"type":"error","error":{"type":"overloaded_error",…}}`. Classification there runs through `classifyStatusType(0, type, message)` — category is driven by the error `type` alone, since there is no HTTP status to key on — which is a distinct code path from the non-2xx `classifyHTTP` route that the existing error-mapping test (`anthropic_test.go`, using `w.WriteHeader`) exercises.

End state: a clearly-named Anthropic adapter test drives a fake server that responds **200** with `Content-Type: text/event-stream` and emits an `event: error` frame (e.g. `overloaded_error`, and at least one other such as `rate_limit_error` to show the `type`-keyed mapping), then asserts the turn surfaces through `Stream.Err()` as a provider `*agentkit.Error` whose `Category` matches the mapped sentinel via `errors.Is` (`ErrOverloaded` / `ErrRateLimited`), whose `Type` carries the provider error-type string, and whose `Raw` is the verbatim error frame bytes. No production code changes are expected — `classifyStreamError` already implements the behavior; this phase pins it.

**Done when:** R-FR35-46U7 is covered by a clearly-named test (Anthropic 200 + `text/event-stream` emitting `event: error`, asserting the mapped `Category` via `errors.Is`, the populated `Type`, and the byte-for-byte `Raw`); the existing D7 ids remain green; suite green per design Conventions.
