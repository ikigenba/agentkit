# Decision 21 — The shared retry executor (`internal/retry`)

**Decision.** The full-jitter exponential-backoff retry *mechanism* lives once, in a new leaf package `internal/retry`. D11 owns the retry *policy contract* (the retryable category set, the per-round-trip budget scope, the no-retry-after-first-byte streaming rule); **D21 owns the single implementation** that every retrying call site drives. Before D21 the identical mechanism — `Clock`/`realClock`, the defaults `4 / 500ms / 30s`, `backoffCap`, the `MaxElapsed`-bounding, the `Retry-After`-vs-jitter delay choice, and the 5-sentinel retryable classifier shape — was copied **four** times: root (`retry.go` + `orchestration.go`), `internal/openaicompat`, `google`, and a fourth hand-rolled loop in `mcp.go` `discoverMCPTools`. D19 already *claimed* embeddings "reuse the D11 executor"; they did not, because root's executor was unexported. D21 makes that claim true.

**`internal/retry` is a pure leaf — it imports only the standard library.** This is load-bearing and corrects the surface intuition that the executor could import root for `RetryPolicy`/`*Error`/the D7 sentinels: root is itself a primary caller (the chat tool loop in `orchestration.go` **and** MCP discovery in `mcp.go` both retry), so root imports `internal/retry`; therefore `internal/retry` must not import root, or the graph cycles. The D7-specific knowledge (which error categories are retryable, how to read `*Error.RetryAfter`) stays in `agentkit` and is **injected** into the executor as a `classify` function. Every sub-package in the module may import `internal/retry` (it is module-internal, not consumer-importable) — the chat orchestrator, the MCP discovery loop, and both embedding adapters all call the same `Do`.

```go
package retry // internal/retry — stdlib only

// Clock is the deterministic time seam: tests inject a fake to assert attempt
// counts and honored delays without real sleeps. It is the single Clock for the
// whole module (the chat orchestrator also reads Now() off it for log timestamps).
type Clock interface {
    Now() time.Time
    Sleep(ctx context.Context, d time.Duration) error // honors ctx; returns ctx.Err() on cancel
    Jitter(cap time.Duration) time.Duration           // uniform in [0,cap); 0 when cap<=1
}

// RealClock is the production Clock (wall time, real sleeps, math/rand jitter).
type RealClock struct{}

// Policy is the plain value-mirror of agentkit.RetryPolicy, decoupling the
// executor from root. Zero fields default inside Do: MaxAttempts→4,
// BaseDelay→500ms, MaxDelay→30s, MaxElapsed→0 (no overall cap).
type Policy struct {
    MaxAttempts      int
    BaseDelay        time.Duration
    MaxDelay         time.Duration
    MaxElapsed       time.Duration
    IgnoreRetryAfter bool
}

// Decision is how the caller classifies one attempt's error. Retryable=false
// (or attempt==MaxAttempts) ends retrying. RetryAfter>0 is used as the next
// delay (unless Policy.IgnoreRetryAfter); otherwise full-jitter backoff applies.
type Decision struct {
    Retryable  bool
    RetryAfter time.Duration
}

// Do runs attempt under the policy. On a retryable error it waits
// min(server-or-jittered-backoff, remaining MaxElapsed budget) on clk, then
// retries; it stops at MaxAttempts, on a non-retryable Decision, when the next
// delay would exceed the MaxElapsed budget, or on ctx cancellation — surfacing
// the last attempt error (or ctx.Err()). onRetry, when non-nil, fires once
// before each backoff sleep (the chat path logs a "retry" record there).
func Do[T any](
    ctx context.Context,
    p Policy,
    clk Clock,
    attempt func() (T, error),
    classify func(error) Decision,
    onRetry func(err error, delay time.Duration),
) (T, error)
```

Settled choices:
- **Leaf package, injected classification.** `internal/retry` depends on nothing in the module; the D7 retryable-set and `*Error.RetryAfter` reading live in `agentkit` as the `classify` closure passed to `Do`. This keeps the dependency graph acyclic (root → `internal/retry`, never the reverse) and keeps D7's error taxonomy the single owner of "what is retryable."
- **Defaults live in `internal/retry`.** The values `4 / 500ms / 30s` are applied inside `Do` against zero `Policy` fields. `agentkit.RetryPolicy` (D11) remains the consumer-facing type with its documented "0 → default" semantics; each call site does a trivial five-field copy `agentkit.RetryPolicy → retry.Policy`. The default *constants* now have exactly one home; root's `retry.go` defaults block and the two adapters' `embeddingRetryDefaults` copies are deleted.
- **Generic `Do[T]`, not `any`.** Chat returns `*agentkit.RoundTrip`, MCP discovery returns `[]mcp.Tool`, embeddings return a `(vectors, usage)` pair — the type parameter avoids assertions at every call site (idiomatic Go 1.26).
- **One `Clock`, shared with logging.** The `Clock` abstraction relocates here from root's `retryClock`. `Conversation.retryClock` is retyped `retry.Clock`; `log.go`'s `logNow()` keeps reading `.Now()` off the same injected clock. **Logging does not move into `internal/retry`** — the executor knows nothing about `LogRecord`; the chat call site supplies an `onRetry` that calls `s.logError(c, "retry", err)`. The three duplicate fake clocks across the test suites collapse onto one `retry.Clock` fake.
- **`internal/openaicompat`'s exported `EmbeddingRetryClock` is removed** in favor of `retry.Clock`; its `Config.Clock` field is retyped. The `google` adapter's `embeddingRetryClock`/`realEmbeddingRetryClock` and all the `embedding*` delay helpers are deleted and replaced by a `retry.Do` call.
- **The four call sites become thin adapters over `Do`:**
  - chat `roundTripWithRetry` — `attempt` issues `Provider.RoundTrip` and lifts `rt.Err()`/`rt==nil` into the error; `classify` is the D7 closure; `onRetry` logs.
  - `mcp.go discoverMCPTools` — `attempt` runs `client.ListTools` and maps the error via `mcpError(serverName, …)`; `classify` is the same D7 closure (so MCP discovery's transient-retry / fail-fast behavior is unchanged).
  - `google` / `openaicompat` embeddings — `attempt` is `embedChunk`; `classify` is the D7 closure; `onRetry` nil.
- **No behavior change at any call site.** D11's and D19's existing ids (R-P3LQ-QY2X, R-P4TN-4PTM, R-P61J-IHKB, R-Y878-6UDJ, R-P79F-W9B0, R-P8HC-A11P, R-6XDZ-VW6L, R-6YLW-9NXA, R-YO2D-MV88) continue to assert the policy end-to-end through chat, MCP, and embeddings unchanged. D21's own ids prove the *extracted unit* directly with a fake clock — the layer those end-to-end ids previously could not isolate.

**Rejected.**
- *Executor imports root for `RetryPolicy`/`*Error`/the sentinels (the eval-report sketch)* — produces an import cycle, since root imports the executor for the chat and MCP retry loops. Rejected on compile grounds; classification is injected instead.
- *Keep the executor in root and have the embedding adapters import root's exported `Do`* — needs no new package, but it pushes retry plumbing and an internal test-only `Clock` type onto root's consumer-facing surface. A leaf `internal/retry` keeps the mechanism off the public API, matching `internal/httpx` and `internal/sse`.
- *Non-generic `Do` returning `any`* — forces a type assertion at all four call sites; the generic is cleaner and type-safe.
- *Fold the mechanism into D11 rather than a new Decision* — D11 owns the policy contract and its end-to-end proofs; conflating the shared-implementation seam with it would muddy two separable concerns (mirrors D9-seam vs D10-orchestration and D18-surface vs D19-SPI). D11 and D19 are instead rewritten in place to reference D21.

**Verification.**
- R-IUBG-95CC — `Do`, given a `classify` that marks errors retryable and **no** server `RetryAfter`, retries on a fake `Clock` with full-jitter exponential backoff whose per-attempt cap doubles `BaseDelay` up to `MaxDelay` (assert the cap sequence the injected `Jitter` is asked for, e.g. base, 2×base, … clamped at `MaxDelay`); when a `Decision` carries `RetryAfter > 0` that exact delay is used instead, and with `Policy.IgnoreRetryAfter = true` the server delay is ignored and jittered backoff is used. The discriminating property is the honored delay values, not merely that a retry occurred.
- R-IWR9-0OTQ — `Do` makes at most `MaxAttempts` attempts then surfaces the **final** attempt's error; a `classify` returning `Retryable:false` ends retrying after the first attempt and performs **no** `Clock.Sleep` call (asserted via the fake clock recording zero sleeps), surfacing that error immediately.
- R-IXZ5-EGKF — with `MaxElapsed` set, when the next computed delay would carry total elapsed past the budget (fake clock advanced past `MaxElapsed - elapsed`), `Do` stops retrying and surfaces the last error rather than sleeping beyond the budget; with `MaxElapsed == 0` no overall cap is applied and retrying proceeds to `MaxAttempts`.
- R-IZ71-S8B4 — context cancellation during a backoff sleep (the fake `Clock.Sleep` returns `ctx.Err()`) ends retrying promptly and `Do` returns the context error, not the attempt error and not a further attempt.
- R-J0EY-601T — a `retry.Policy` with zero `MaxAttempts`/`BaseDelay`/`MaxDelay` is executed by `Do` as `4 / 500ms / 30s` respectively (assert via attempt count reaching 4 and the first backoff cap equal to 500ms on the fake clock), proving the default constants live solely in `internal/retry`; a `MaxElapsed` of zero remains "no cap".
